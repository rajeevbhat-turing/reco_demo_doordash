# Merchant LocalStorage Structure

This document describes the localStorage structure used for merchant data persistence.

## Overview

All merchant data is stored under the `merchant.*` namespace in localStorage. This provides a clear separation from other application data and makes it easy to manage merchant-specific data.

## Storage Keys

### Store-Level Keys (Zustand Stores)

| Key | Description | Store File |
|-----|-------------|------------|
| `merchant.home` | Home page data (metrics, dashboards) | `merchant-home-store.ts` |
| `merchant.orders` | Orders data and filters | `merchant-orders-store.ts` |
| `merchant.menu` | Menu categories, items, and settings | `merchant-menu-store.ts` |
| `merchant.users` | User management data | `merchant-users-store.ts` |
| `merchant.settings` | Account and store settings | `merchant-settings-store.ts` |
| `merchant.financials` | Financial data (transactions, statements) | (Future) |
| `merchant.customers` | Customer insights and reviews | (Future) |
| `merchant.marketing` | Marketing campaigns and loyalty | (Future) |
| `merchant.store-availability` | Store hours and availability settings | `merchant-store-availability-store.ts` |
| `merchant.request-delivery` | Request delivery form data | (Future) |

### Component-Level Keys (Component State)

Component-level state uses the pattern: `merchant.{section}.{component}.{field}`

**Examples:**
- `merchant.users.form.role` - User role selection in form
- `merchant.users.form.firstName` - First name input
- `merchant.users.search.query` - Search query
- `merchant.orders.filters.searchQuery` - Orders search
- `merchant.orders.tabs.activeTab` - Active tab (Active/Scheduled/History)
- `merchant.menu.filters.searchQuery` - Menu search
- `merchant.menu.filters.selectedFilter` - Menu filter selection
- `merchant.settings.store.storeName` - Store name
- `merchant.settings.modals.isStoreNameOpen` - Modal open state
- `merchant.settings.account.autoGeneratedDescriptions` - Toggle state

## Usage

### Using Stores

All merchant stores use Zustand with the `persist` middleware for automatic localStorage synchronization:

```typescript
import { useMerchantMenuStore } from '@/store/merchant'

function MyComponent() {
  const { categories, updateItemStatus } = useMerchantMenuStore()
  
  // Store automatically persists to localStorage
  return (
    // Your component
  )
}
```

### Persisting Component-Level State

For component-level state (form fields, UI state, filters, etc.), use the `useMerchantPersistedState` hook:

```typescript
import { useMerchantPersistedState } from '@/lib/hooks/useMerchantPersistedState'

function ManageUsersPage() {
  // Persists to: merchant.users.form.role
  const [selectedRole, setSelectedRole] = useMerchantPersistedState(
    'users',      // section
    'form',       // component
    'role',       // field
    ''            // default value
  )
  
  // Persists to: merchant.users.search.query
  const [searchQuery, setSearchQuery] = useMerchantPersistedState(
    'users',
    'search',
    'query',
    ''
  )
}
```

**Storage Key Structure:**
- Format: `merchant.{section}.{component}.{field}`
- Example: `merchant.users.form.role`
- Example: `merchant.orders.filters.searchQuery`
- Example: `merchant.settings.modals.isStoreNameOpen`

### Using Storage Utilities

For direct localStorage access, use the utility functions:

```typescript
import { 
  MerchantStorageKeys, 
  getMerchantStorage, 
  setMerchantStorage 
} from '@/lib/utils/merchant-storage'

// Get data
const data = getMerchantStorage(MerchantStorageKeys.HOME, defaultValue)

// Set data
setMerchantStorage(MerchantStorageKeys.HOME, newData)
```

## Migration from Old Keys

If you have existing data stored under old keys (e.g., `merchantstore.menu`), you may need to migrate:

1. Old keys: `merchantstore.menu`, `merchantstore.users`
2. New keys: `merchant.menu`, `merchant.users`

The stores have been updated to use the new keys automatically. Old data will remain in localStorage but won't be accessed by the new stores.

## Best Practices

1. **Always use the store hooks** - Don't directly access localStorage for merchant data
2. **Use the storage keys constant** - Don't hardcode storage key strings
3. **Handle errors gracefully** - The utility functions include error handling
4. **Keep stores focused** - Each store should handle one domain of merchant data
5. **Use TypeScript types** - All stores export TypeScript interfaces for type safety

## Clearing Data

To clear all merchant data:

```typescript
import { clearAllMerchantStorage } from '@/lib/utils/merchant-storage'

clearAllMerchantStorage()
```

Or clear a specific store:

```typescript
import { removeMerchantStorage, MerchantStorageKeys } from '@/lib/utils/merchant-storage'

removeMerchantStorage(MerchantStorageKeys.MENU)
```
